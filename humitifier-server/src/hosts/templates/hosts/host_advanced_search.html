Query Syntax Help

Basic Structure

Queries consist of field comparisons combined with logical operators.

field_name operator value

Comparison Operators

  • = — equals (case-insensitive for strings)
  • > — greater than
  • >= — greater than or equal
  • < — less than
  • <= — less than or equal
  • contains — substring match (case-insensitive)

Values

  • Strings: "value" or 'value' (use quotes for text)
  • Numbers: 42 or 3.14 (no quotes)
  • Booleans: true or false (no quotes)
    • {% if layout.wild_wasteland %}
  • Capes: no capes!
    • {% endif %}

Logical Operators

  • AND — both conditions must be true
  • OR — at least one condition must be true

Grouping with Brackets

Use brackets { and } to control evaluation order.

{ condition1 OR condition2 } AND condition3

Aggregations for Array Fields

Apply aggregation functions to array fields using the syntax: function(field[])

  • min(field[]) — minimum value (integer fields)
  • max(field[]) — maximum value (integer fields)
  • sum(field[]) — sum of values (integer fields)
  • concat(field[]) — concatenate strings (string fields)
  • count(field[]) — count array elements (all array fields)

Examples

Find Ubuntu hosts:

facts.os.name = "Ubuntu"

Find hosts with 4 or more CPUs:

facts.cpu.count >= 4

Find Ubuntu hosts with 4+ CPUs:

facts.os.name = "Ubuntu" AND facts.cpu.count >= 4

Find either Ubuntu or Debian hosts:

facts.os.name = "Ubuntu" OR facts.os.name = "Debian"

Find Linux hosts with "web" in hostname:

{ facts.os.name = "Ubuntu" OR facts.os.name = "Debian" } AND facts.hostname contains "web"

Find hosts with more than 100 packages:

count(facts.packages[].name) > 100

Find hosts with max memory size greater than 8GB:

max(facts.memory[].size) >= 8192
